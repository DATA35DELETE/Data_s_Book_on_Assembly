Assembly : Overview

What is Assembly?

 Assembly is a low-level programming language to give instruction in the language and communicate with your local machine known as architecture's machine code instructions. Assembly language usually has a statement or machine code instruction for each machine action. and theres constants, comments, assembler directives, symbolic labels such as memory locations, registers and macros are generally also supported. that means you can talk with your own machine using this language and you can literally use it for making or doing everything.

Brief History:

The first assembly code which is used to represent a machine code instructions is found in Kathleen and Andreww Donald Booth's  1947 work, Coding for A.R.C. Assembly code is converted into executable machine code by a utility program called "Assembler". The Term "Assembler" is generally used in to Wilkes, Wheeler and Gill in their 1951 book The Preparation of Programs for an Electronic Digital Computer,

What is an "Assembler?"

 the "Assembler" term means that the program that assembles another program consisting of several sections into a single program. the conversion process knowns as the term assembly, as in assembling the source code. The computational step when an assembler is processing a program called assembly time.

Its relation to machine code:

Assembly code relates to machine code and give that machine instructions via machine code. 
theres a lot of assembly languages each one specific to a particulat computer architecture such as x86 or ARM

    CPU stands for "Central Processing Unit" know as Central processor or just processor. Its electronic circuitry executes instructions of a computer program. Every operation, algorithm, arithmetics, you inputs and the programs outputs, etc. is being handled in this unit of the computer.     What is RAM (Random Access Memory)?

    RAM stands for Random Access Memory meaning you can access the memory like, reading, removing (freeing) the stored memories in it, update, write into it and expand the space of the memory stored in it. You are probably familiar with this structure if you ever got into a low-level programming language before. but RAM alone itself can't do every function we mentioned itself. Thats where the CPU comes in. CPU uses a structure called System Bus Structure. Basically theres a Memory Controller which operates the entire memory between CPU and RAM are synchronized. it was located in the motherboard on older computers but of course, nowadays they just integrate it in the CPU itself for better speed and efficiency. The CPU takes the memories data, memories memory adresses and control checks via System Bus. The memory address is a address in RAM which equals to a value in it and via data bus in System Bus Structure. When the CPU needs to access a memory, it places the address of the memory on the address bus and sends a signal via control bus to say that if its reading or writing the address. If it reads, the memory controller retrieves the data by looking at the specfied address. then sends the data with the data bus back to the CPU to process it. If it writes, the CPU just sends the data via data bus to store the data at the designated memory location. This process is repeated billions of times in the CPU So continouing on:
