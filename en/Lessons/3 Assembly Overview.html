Assembly : Overview

What is Assembly?

 Assembly is a low-level programming language to give instruction in the language and communicate with your local machine known as architecture's machine code instructions. Assembly language usually has a statement or machine code instruction for each machine action. and theres constants, comments, assembler directives, symbolic labels such as memory locations, registers and macros are generally also supported. that means you can talk with your own machine using this language and you can literally use it for making or doing everything.

Brief History:

The first assembly code which is used to represent a machine code instructions is found in Kathleen and Andreww Donald Booth's  1947 work, Coding for A.R.C. Assembly code is converted into executable machine code by a utility program called "Assembler". The Term "Assembler" is generally used in to Wilkes, Wheeler and Gill in their 1951 book The Preparation of Programs for an Electronic Digital Computer,

What is an "Assembler?"

 the "Assembler" term means that the program that assembles another program consisting of several sections into a single program. the conversion process knowns as the term assembly, as in assembling the source code. The computational step when an assembler is processing a program called assembly time.

Its relation to machine code:

Assembly code relates to machine code and give that machine instructions via machine code. 
theres a lot of assembly languages each one specific to a particulat computer architecture such as x86 or ARM

Comparison with high-level languages:

The difference between high-level languages such as python, go, ruby, C#, PHP, JavaScript, etc. and Assembly is the same with a car which is automatic on gear and another which is manual meaning you have to calculate every gear movement by hand and calculate every possible combination while you drive.
So in coding wise: if you do anything in high-level language its proably being processed in the backend that includes as simply as even creating a new variable. Somewhere in the back of your computer theres a machine instruction that tells your RAM to open enough space for that memory to store the variable. But in Assembly you do everything manually. byte by byte, bit by bit.

    CPU and Memory Basics

    What is a CPU?

    CPU stands for "Central Processing Unit" know as Central processor or just processor. Its electronic circuitry executes instructions of a computer program. Every operation, algorithm, arithmetics, you inputs and the programs outputs, etc. is being handled in this unit of the computer. This unit is blends in overtime with external components like main memory, and I/O circuitry, and specialized coprocessors for example Graphics Processor Unit (GPU). almost like anything else the CPU was never always the same, it changed over time, but their fundamental operation remains almost unchanged.
